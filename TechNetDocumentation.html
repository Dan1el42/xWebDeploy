Introduction

The xWebDeploy module is a part of the Windows PowerShell Desired State Configuration (DSC) Resource Kit, which is a collection of DSC Resources produced by the PowerShell Team. This module contains the xWebPackageDeploy and xWebDeploy resources. This DSC Resource allows you to install an IIS site using WebDeploy IIS extension (MSWebDeploy version 3.5).

All of the resources in the DSC Resource Kit are provided AS IS, and are not supported through any Microsoft standard support program or service. The ""x" in xWebDeploy stands for experimental , which means that these resources will be fix forward and monitored by the module owner(s).

Please leave comments, feature requests, and bug reports in the Q & A tab for this module.

If you would like to modify xWebDeploy module, feel free. When modifying, please update the module name, resource friendly name, and MOF class name (instructions below). As specified in the license, you may copy or modify this resource as long as they are used on the Windows Platform.

Installation

To install xWebDeploy module

  • Unzip the content under $env:ProgramFiles\WindowsPowerShell\Modules folder

To confirm installation:

  • Run Get-DSCResource to see that xWebDeploy is among the DSC Resources listed

Requirements

This module requires the latest version of PowerShell (v4.0, which ships in Windows 8.1 or Windows Server 2012 R2). To easily use PowerShell 4.0 on older operating systems, install WMF 4.0. Please read the installation instructions that are present on both the download page and the release notes for WMF 4.0.

Description

The xWebDeploy module contains the xWebDeploy DSC Resource. This DSC Composite Configuration to install WebDeploy 3.5 IIS extension using DSC builtin resource Package and deploy an IIS site from a given web package that was exported from reference server using MSWebDeploy 3.0 or 3.5. It supports Web Deploy sync, dump verbs and iisApp and contentPath parameters.

The xWebDeploy module also contains the xWebPackageDeploy DSC Resource. This DSC Resource to install an IIS site from a given web deploy package formated in zip file. 

Details

xWebDeploy composite resource has following properties:

  • SourcePath: The full path on the filesystem were the web deploy package(in zip file) is located.
  • Destination: The IIS content path or IIS site name to install the WebDeploy Zip pacakge.
  • WebDeployMsi: The uri to download the WebDeploy version 3.5.

xWebPackageDeploy resource has following properties:

  • SourcePath: The full path on the filesystem were the web deploy package(in zip file) is located.
  • Destination: The IIS content path or IIS site name to install the WebDeploy Zip pacakge.
  • Ensure: Only Present or Absent are allowed.

Renaming Requirements

When making changes to these resources, we suggest the following practice:

  1. Update the following names by replacing the "x" with "c" (short for "Community") or another prefix of your choice:
    • Module name (ex: xWebDeploy becomes cWebDeploy)
    • Resource folder (ex: xWebDeploy becomes Contoso_cWebDeploy)
    • Resource Name (ex: xWebDeployPackage becomes Contoso_cWebDeployPackage)
    • Resource Friendly Name (ex: xWebPackageDeploy becomes cWebPackageDeploy)
    • MOF class name (ex: xWebPackageDeploy becomes Contoso_cWebPackageDeploy)
    • Filename for the <resource>.schema.mof (ex: xWebPackageDeploy .schema.mof becomes Contoso_cWebPackageDeploy.schema.mof)
  2. Update module and metadata information in the module manifest
  3. Update any configuration that use these resources

We reserve resource and module names without prefixes ("x" or "c") for future use (e.g. "MSFT_WebDeploy" or "WebDeploy"). If the next version of Windows Server ships with a "WebDeploy" resource, we don't want to break any configurations that use any community modifications. Please keep a prefix such as "c" on all community modifications.
